Write clean, maintainable code following these rules:

- Use descriptive variable and function names.
- Keep functions under 20 lines when possible.
- One function should perform one responsibility.
- Follow DRY(Don't Repeat Yourself) and KISS(Keep it Simple,Stupid) principles.
- Avoid unnecessary comments; comment only complex logic.
- Use constants instead of magic numbers.
- Validate inputs early with guard clauses.
- Handle errors properly.
- Keep consistent formatting and naming.
- Remove unused code and imports.
- Separate business logic from UI and database logic.
- Write readable code over clever code.
